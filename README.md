## Базовая часть задания

### 1. Настройка Git и репозитория
- Создан личный репозиторий на GitHub
- Освоены базовые команды Git
- Регулярно фиксируются изменения с осмысленными сообщениями

### 2. Написание документов в Markdown
- Все материалы проекта оформлены в формате Markdown
- Изучен синтаксис Markdown
- Подготовлены необходимые документы

### 3. Создание статического веб-сайта
Сайт включает следующие разделы:
- Домашняя страница с аннотацией проекта
- Страница "О проекте" с описанием
- Раздел "Участники" с описанием личного вклада
- Раздел "Журнал" с постами о прогрессе
- Страница "Ресурсы" со ссылками на материалы

### 4. Взаимодействие с организацией-партнёром
- Планируется взаимодействие через куратора проекта
- Будет организована встреча или стажировка
- Будет подготовлен отчёт о взаимодействии

### 5. Отчёт по практике
- Будет составлен на основании шаблона
- Размещён в папозитории в папке reports
- Будет представлен в форматах DOCX и PDF

## Структура репозитория

- `docs/` - документация по практике
- `reports/` - отчёты в формате PDF
- `site/` - статический сайт проекта
- `task/` - текст задания и шаблон отчёта

## Запуск сайта

1. Откройте терминал
2. Перейдите в директорию `site` командой `cd site`
3. Запустите локальный веб-сервер командой `python3 -m http.server 3000`
4. Откройте в браузере адрес `http://localhost:3000`

Примечание: Если порт 3000 занят, можно использовать другие порты (например, 4000, 5000 или 9000).
