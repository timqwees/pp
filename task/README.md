# Задание по проектной практике

## Базовая часть

### 1. Настройка Git и репозитория
- Создание личного или группового репозитория на GitHub или GitVerse
- Освоение базовых команд Git: клонирование, коммит, пуш и создание веток
- Регулярная фиксация изменений с осмысленными сообщениями к коммитам
- Ожидаемое время: 5 часов

### 2. Написание документов в Markdown
- Оформление всех материалов проекта в формате Markdown
- Изучение синтаксиса Markdown
- Подготовка необходимых документов
- Ожидаемое время: 5 часов

### 3. Создание статического веб-сайта
- Использование HTML и CSS для создания сайта
- Опционально: применение генераторов статических сайтов (Hugo)
- Создание уникального сайта (более 50% уникального контента)
- Включение следующих разделов:
  - Домашняя страница с аннотацией проекта
  - Страница "О проекте" с описанием
  - Раздел "Участники" с описанием личного вклада
  - Раздел "Журнал" с минимум тремя постами о прогрессе
  - Страница "Ресурсы" со ссылками на материалы
- Оформление страниц графическими материалами и медиа-контентом
- Ожидаемое время: 14-22 часа

### 4. Взаимодействие с организацией-партнёром
- Организация взаимодействия через куратора проекта
- Участие в профильных мероприятиях
- Написание отчёта о взаимодействии
- Ожидаемое время: 8 часов

### 5. Отчёт по практике
- Составление отчёта на основании шаблона
- Размещение в репозитории в папке reports
- Представление в форматах DOCX и PDF
- Загрузка в СДО (LMS)

## Вариативная часть

### Создание PHP фреймворка
В рамках вариативной части мы разработаем собственный PHP фреймворк, вдохновленный принципами работы React/JSX. 

#### Пример реализации
Для демонстрации работы фреймворка создан пример списка задач (Todo List), который показывает основные возможности:
- Компонентный подход
- Управление состоянием
- Рендеринг HTML
- Взаимодействие с API

Пример находится в директории `examples/`:
- `todo.php` - основной файл с компонентом списка задач
- `api.php` - API для обработки действий

Чтобы запустить пример:
1. Установите зависимости: `composer install`
2. Запустите PHP-сервер: `php -S localhost:8000 -t examples`
3. Откройте в браузере: `http://localhost:8000/todo.php`

#### Цели проекта
1. Создать легковесный PHP фреймворк с компонентным подходом
2. Реализовать систему шаблонизации, похожую на JSX
3. Разработать систему управления состоянием
4. Создать виртуальный DOM для PHP

#### Технический стек
- PHP 8.x
- Composer для управления зависимостями
- PHPUnit для тестирования
- PHP Parser для работы с AST

#### Основные компоненты
1. **Компонентная система**
   - Создание и управление компонентами
   - Система пропсов и состояний
   - Жизненный цикл компонентов

2. **Шаблонизатор**
   - Синтаксис, похожий на JSX
   - Поддержка условных конструкций
   - Поддержка циклов
   - Поддержка компонентов

3. **Виртуальный DOM**
   - Представление DOM в PHP
   - Дифференциальный алгоритм
   - Эффективное обновление DOM

4. **Система управления состоянием**
   - Централизованное хранилище
   - Подписка на изменения
   - Обновление компонентов

#### План разработки
1. **Этап 1: Исследование**
   - Анализ существующих решений
   - Проектирование архитектуры
   - Определение API

2. **Этап 2: Базовая реализация**
   - Создание компонентной системы
   - Реализация шаблонизатора
   - Разработка виртуального DOM

3. **Этап 3: Расширение функционала**
   - Добавление системы состояний
   - Реализация событий
   - Оптимизация производительности

4. **Этап 4: Документация и тестирование**
   - Написание технической документации
   - Создание примеров использования
   - Написание тестов

#### Ожидаемые результаты
1. Рабочий PHP фреймворк с компонентным подходом
2. Подробная техническая документация
3. Примеры использования
4. Набор тестов
5. Демонстрационное приложение

#### Критерии оценки
1. Функциональность и работоспособность
2. Качество кода и архитектуры
3. Полнота документации

5. Качество тестового покрытия
